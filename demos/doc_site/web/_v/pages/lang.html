PHP环境中如果配置了 APC，则会使用 APC 缓存语言数据

多语言解决方案由函数 L 完成

/** * 调用语言处理函数，将一段文本转换成对应的语言文本 * @param string $text 要转换的文本 * @param string or array $langs 要转换的目标语言，可以用数组方式传入多个 * @return string 对应的语言文本 */ function L( $text, $langs=null ) { return MSVC::getLang( $text, $langs ); }

在 View 层调用多语言使用{#xxxx}，如果没有找到匹配的语言将会输出原始内容

在 View 层调用或使用 L 函数时未指定要转换的语言，将会读取HTTP头信息中的ACCEPT_LANGUAGE，如果是5字节的语言则会首先尝试匹配5字节的语言文件，之后匹配2字节的语言文件 如['zh-cn','zh']

/** * 调用语言处理函数，将一段文本转换成对应的语言文本 * @param string $text 要转换的文本 * @param string or array $langs 要转换的目标语言，可以用数组方式传入多个 * @return string 对应的语言文本 */ function L( $text, $langs=null ) { return MSVC::getLang( $text, $langs ); }

多语言数据由配置LANG_PATH指定，默认为项目根目录下的langs

多语言数据文件可以是2字节的如zh，也可以是5字节的如zh-cn，全部小写

多语言文件示例

{$zh_langs}
